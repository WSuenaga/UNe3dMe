# How to Add a New Method

## 1．概要

本ページでは，UNe3dMe に新しい 3 次元再構築手法を追加する手順を説明する．  
対象読者は，新規手法の実装ラッパ追加，UI への統合，viewer・export・evaluation への接続を行う開発者である．

## 2．追加対象の整理

新しい手法を追加する前に，以下を整理する．

- 手法名
- 手法カテゴリ
  - NeRF 系
  - Gaussian Splatting 系
  - 3sters 系
  - VGGT 系
  - mds 系
  - その他
- 実行に必要な環境
- 期待する入力
- 出力 3D モデルの形式


## 3．実装追加の全体手順

追加作業の基本的な流れは以下のとおりである．

1. 手法実装本体を `models/` または対応ディレクトリへ配置する．
2. 必要に応じて `scripts/` に補助スクリプトを追加する．
3. `methods.py` に実行ラッパ関数を追加する．
4. 必要なら export 関数を追加する．
5. 必要なら render / evaluation 関数を追加する．
6. `demo.py` に UI 入力欄とイベント接続を追加する．
7. viewer で利用する場合は viewer 連携を追加する．
8. ドキュメントを更新する．

## 4．手法実装の配置

### 4.1 `models/` に配置する場合
- 外部リポジトリや submodule として管理する実装を配置する．

### 4.2 `scripts/` に配置する場合
- 実行補助スクリプト
- 前後処理スクリプト
- 変換スクリプト
- viewer 用スクリプト

を配置する．

## 5．`methods.py` への追加

### 5.1 基本方針

`methods.py` には，UI 固有の処理を書かず，実行ラッパとして実装する．  
主に以下を担当する．

- 引数の受け取り
- コマンドライン組み立て
- 出力先決定
- local / slurm 切替
- subprocess 実行
- 結果返却

### 5.3 render / evaluation 関数のテンプレート

## 6．出力先規約の設計

新規手法追加時は，出力先ディレクトリ規約を明確にする必要がある．

確認する項目は以下のとおりである．

- どのカテゴリ配下へ保存するか
- config，checkpoint，render，evaluation をどこへ保存するか
- viewer が参照するファイルをどこへ置くか，あるいはどこに出力されるか．

### 6.1 例

```text
outputs/
└── <method_name>/
    └── <dataset_name>/
        ├── results/
        ├── renders/
        └── scene.ply
```

## 7．local / slurm 対応

### 7.1 local 実行
- ローカル環境で直接コマンドを実行する．

### 7.2 slurm 実行
- 必要に応じて sbatch スクリプトや専用ラッパへ委譲する．

### 7.3 実装上の注意
- mode は `local` または `slurm` のみを許可する．
- 両対応しない手法では，未対応モードを明示的にエラーとする．

## 8．`demo.py` への UI 追加

新しい手法を UI から利用できるようにするには，`demo.py` 側の変更も必要である．

### 8.1 追加するもの
- 手法選択 UI
- 手法固有パラメータ入力欄
- 実行ボタン
- 出力表示欄
- ログ表示欄

### 8.2 イベント接続
- ボタン押下時に `methods.recon_<method_name>()` を呼び出す．
- 必要に応じて export / render_eval 関数も接続する．

## 9．viewer 対応

viewer で可視化したい場合は，以下を確認する．

- 出力形式が既存 viewer で読めるか
- 新しい viewer スクリプトが必要か
- `local_backend.viewer()` から呼び出せるか

## 10．evaluation 対応

評価に対応させる場合は，以下を確認する．

- レンダリング画像を生成できるか
- GT 画像との対応関係を構築できるか
- 評価関数へ入力可能な形式か
- 結果保存先が規約に従っているか

## 11．翻訳対応

UI 文言を追加する場合は，翻訳対応も必要である．

確認する項目は以下のとおりである．

- 手法名の表示ラベル
- パラメータ説明文
- 核種コンポーネント文言
- ログメッセージ
- エラーメッセージ

必要に応じて，翻訳ファイルおよび UI 更新処理を修正する．

## 12．テスト項目

新しい手法を追加した後は，少なくとも以下を確認する．

- UI から実行できるか
- local 実行が正常に動くか
- 必要なら slurm 実行が正常に動くか
- 出力先が想定どおりか
- ログが保存されるか
- viewer 連携が必要なら正常に表示できるか
- render / evaluation が必要なら正常に動くか
- エラー時に UI へ適切なメッセージが返るか

## 13．ドキュメント更新箇所

新しい手法を追加した際は，少なくとも以下のドキュメントを更新する．

- Welcome page の対応手法一覧
- Installation
- Developer Guides 内の関連ページ
